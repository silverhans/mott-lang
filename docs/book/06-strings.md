# Глава 6. Строки

Строки в Mott называются **`deshnash`** (чеч. "слова" — множественное от *дош*). Это UTF-8 последовательности байтов, могут хранить любой текст на любом языке.

## Литералы

```mott
xilit s: deshnash = "salam"
xilit empty: deshnash = ""
xilit chechen: deshnash = "Маршалла хьо"
xilit emoji: deshnash = "привет 👋"
```

Кавычки — **только двойные** (`"..."`). Одинарных нет.

## Escape-последовательности

Внутри строкового литерала следующие комбинации заменяются на спец-символы:

| Что написать | Что получится |
|---|---|
| `\n` | перевод строки |
| `\t` | табуляция |
| `\r` | carriage return |
| `\"` | буквальная кавычка |
| `\\` | буквальный слэш |
| `\{` | буквальная `{` (без интерполяции) |
| `\}` | буквальная `}` |

```mott
yazde("строка 1\nстрока 2")           // две строки
yazde("он сказал: \"salam\"")         // с кавычками внутри
yazde("путь: C:\\User\\hans")         // слэши
```

## Интерполяция

Внутри строки можно вставлять значение переменной через `{имя}`:

```mott
fnc kort() {
    xilit name: deshnash = "Hans"
    xilit age: terah = 30
    yazde("Меня зовут {name}, мне {age}.")
}
// Меня зовут Hans, мне 30.
```

Работает со всеми примитивными типами — числа и bool'ы автоматически конвертируются в строковое представление:

```mott
xilit x: daqosh = 3.14
xilit ok: bool = baqderg
yazde("π ≈ {x}, готов? {ok}")
// π ≈ 3.14, готов? baqderg
```

### Выражения в интерполяции

В `{...}` можно вставлять **произвольные выражения**, не только имена переменных:

```mott
xilit x: terah = 5
xilit y: terah = 3
yazde("sum = {x + y}")                 // sum = 8
yazde("bigger? {x > y}")               // bigger? baqderg
yazde("doubled = {x * 2}")             // doubled = 10
```

Вызовы функций тоже работают:

```mott
fnc square(n: terah) -> terah {
    yuxadalo n * n
}

fnc kort() {
    xilit x: terah = 5
    yazde("x^2 = {square(x)}")         // x^2 = 25
}
```

**Одно ограничение**: внутри `{...}` **нельзя** писать строковые литералы. Такой код сломает грамматику — внутренняя `"` закроет внешнюю строку:

```mott
yazde("hi, {greet("Hans")}")           // ошибка: nested string
```

Обход — положить в переменную:

```mott
xilit name: deshnash = "Hans"
yazde("hi, {greet(name)}")             // ок
```

Пустая интерполяция `{}` — тоже ошибка (ну не зачем так делать).

## Как "выключить" интерполяцию

Если тебе нужна буквальная фигурная скобка в тексте — экранируй её:

```mott
yazde("JSON: \{\"key\": \"value\"\}")
// JSON: {"key": "value"}
```

## Сравнение

`==` и `!=` работают на строках — побайтовое сравнение:

```mott
fnc kort() {
    xilit a: deshnash = "salam"
    xilit b: deshnash = "salam"
    xilit c: deshnash = "Salam"           // заглавная S

    yazde(a == b)                         // baqderg
    yazde(a == c)                         // xarco (регистр важен)
    yazde(a != c)                         // baqderg
}
```

**Лексикографическое сравнение** (`<`, `>`, `<=`, `>=` на строках) пока **не реализовано**. Если попробуешь — компилятор ругнётся "ordering comparison needs numeric operands".

## Длина — `baram`

`baram(s)` возвращает количество **байтов** в строке как `terah`:

```mott
xilit word: deshnash = "hello"
yazde("{baram(word)}")                    // 5

xilit empty: deshnash = ""
yazde("{baram(empty)}")                   // 0
```

**Осторожно**: для ASCII-строк `baram` это длина в символах. Для мультибайтовых (UTF-8) — это длина в **байтах**, не символах:

```mott
xilit word: deshnash = "маршо"            // 5 букв кириллицы
yazde("{baram(word)}")                    // 10 (каждая буква = 2 байта в UTF-8)
```

Работа с символами (кодпоинтами), а не байтами, — отдельная тема, пока не в MVP.

## Конкатенация — пока нет

В отличие от многих языков, **`+` на строках не работает** в MVP:

```mott
xilit greeting = "salam" + "!"           // ошибка: arithmetic type mismatch / non-numeric
```

Вместо этого используй интерполяцию:

```mott
xilit exclaim: deshnash = "!"
xilit greeting: deshnash = "salam{exclaim}"    // "salam!"
```

Если нужно собрать строку из многих частей — делается через промежуточные переменные и интерполяцию. Это чуть громоздко, но работает. В будущих версиях появится либо оператор `+`, либо функция `concat`.

## Пустая строка

```mott
xilit empty: deshnash = ""
yazde(empty)                              // печатает ничего + \n
yazde("{baram(empty)}")                   // 0
nagah sanna empty == "" {
    yazde("пустая")                       // сработает
}
```

## Функции со строками

Передача — прозрачная, как обычный параметр:

```mott
fnc shout(text: deshnash) {
    yazde("{text}!")
}

fnc kort() {
    shout("marsha")                       // marsha!
    shout("vogilla")                      // vogilla!
}
```

Возврат — тоже работает:

```mott
fnc name_for(code: terah) -> deshnash {
    nagah sanna code == 1 {
        yuxadalo "first"
    }
    nagah sanna code == 2 {
        yuxadalo "second"
    }
    yuxadalo "other"
}

fnc kort() {
    yazde(name_for(1))                    // first
    yazde(name_for(2))                    // second
    yazde(name_for(42))                   // other
}
```

## Управление памятью — чуть-чуть детали

Под капотом `deshnash` это C-шная структура `{ data*, len }` — указатель на байты плюс длина. Когда ты создаёшь интерполированную строку, рантайм выделяет память через `malloc`. В MVP она **не освобождается** никогда — программы маленькие, это осознанное упрощение. В v0.3+ планируется нормальное управление памятью (вероятно GC).

Для обычных программ это не важно. Не пиши миллион интерполированных строк в бесконечном цикле — вот и вся забота.

## Попробуй сам

1. Напиши программу, которая определяет является ли введённое слово палиндромом. (Подсказка: проверь `word == reverse_of_word`. Но `reverse_of_word` придётся писать через... а нет, это потребует работы с байтами, которую мы пока не умеем. Попробуй просто сравнить несколько литералов.)
2. Напиши функцию `greet(name: deshnash, hour: terah)` которая пишет "Salam" (утром/днём), "Marsha vogilla" (вечером) в зависимости от часа.
3. Что напечатает `yazde("\\n")`? А `yazde("\n")`? Понимаешь почему разные результаты?
4. Напиши функцию `repeat3(word: deshnash) -> deshnash` — возвращает строку, содержащую слово три раза подряд. Подсказка: используй интерполяцию.
