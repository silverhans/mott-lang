# Глава 5. Функции

Функции — способ упаковать кусок кода в именованный блок, чтобы его можно было вызвать из разных мест. Ты уже видел `fnc kort()` — точку входа. В этой главе разберём всё остальное.

## Объявление

```mott
fnc add(x: terah, y: terah) -> terah {
    yuxadalo x + y
}
```

Разбираем по частям:

- **`fnc`** — объявление функции.
- **`add`** — имя. Любой валидный идентификатор (правила как у переменных).
- **`(x: terah, y: terah)`** — список параметров. У каждого **обязателен тип** — компилятор не выводит типы параметров.
- **`-> terah`** — тип возвращаемого значения.
- **`{ ... }`** — тело.
- **`yuxadalo expr`** — `return` с значением. От чеч. *"юхадало"* — "отдать обратно".

Вызывается обычно:

```mott
fnc kort() {
    xilit sum: terah = add(2, 3)
    yazde("{sum}")       // 5
}
```

## Функции без возврата (void)

Если функция ничего не возвращает, стрелку `-> type` просто не пишешь:

```mott
fnc greet(name: deshnash) {
    yazde("Salam, {name}!")
}

fnc kort() {
    greet("Hans")
    greet("Mansur")
}
```

Такая функция вызывается как **инструкция** (statement), а не выражение. Попытка использовать результат (там где результата нет) даст ошибку:

```mott
xilit x = greet("Hans")     // error: function `greet` returns no value
```

## `yuxadalo` без значения — ранний выход

Ты можешь использовать `yuxadalo` в void-функции чтобы прервать выполнение:

```mott
fnc check(x: terah) {
    nagah sanna x < 0 {
        yazde("отрицательное, пропускаем")
        yuxadalo                  // выйти из функции
    }
    yazde("обрабатываем {x}")
}

fnc kort() {
    check(-5)
    check(10)
}
// отрицательное, пропускаем
// обрабатываем 10
```

В функции с возвратом `yuxadalo` без значения — ошибка.

## Рекурсия

Работает как в любом другом языке. Классический факториал:

```mott
fnc fact(n: terah) -> terah {
    nagah sanna n <= 1 {
        yuxadalo 1
    }
    yuxadalo n * fact(n - 1)
}

fnc kort() {
    yazde("{fact(5)}")    // 120
}
```

**Взаимная рекурсия** (функция A вызывает B, B вызывает A) тоже работает: компилятор сначала собирает все сигнатуры, потом генерирует тела. Порядок объявления в файле не важен.

```mott
fnc is_even(n: terah) -> bool {
    nagah sanna n == 0 {
        yuxadalo baqderg
    }
    yuxadalo is_odd(n - 1)
}

fnc is_odd(n: terah) -> bool {
    nagah sanna n == 0 {
        yuxadalo xarco
    }
    yuxadalo is_even(n - 1)
}

fnc kort() {
    yazde(is_even(4))     // baqderg
    yazde(is_odd(7))      // baqderg
}
```

## Точка входа: `kort`

Функция `kort` — специальная. Это **точка входа** программы, её вызывает рантайм когда ты запускаешь бинарник. Требования:

- **Без параметров**: `fnc kort() { ... }` — OK. `fnc kort(x: terah) { ... }` — ошибка.
- **Без возвращаемого типа**: `fnc kort() -> terah` — ошибка.

`kort` чеченское "голова"; семантика как у `main` в C/Rust/Go.

## Параметры — значения или ссылки?

В Mott параметры передаются **по значению** для примитивов (`terah`, `daqosh`, `bool`) и **по ссылке** (под капотом) для сложных типов (`deshnash`, массивы `[T]`).

Практически это значит: изменения **примитивного** параметра внутри функции **не видны** снаружи:

```mott
fnc try_modify(x: terah) {
    x = 999               // только внутри функции
}

fnc kort() {
    xilit n = 5
    try_modify(n)
    yazde("{n}")          // 5 (не 999)
}
```

Для строк и массивов такой пример написать сложнее (мы это пройдём в главе о массивах), но семантика — "функция видит те же данные, а не копию".

## Сколько параметров

Ограничений сверху нет. Все обязательны (нет опциональных параметров в MVP, нет значений по умолчанию).

```mott
fnc clamp(value: terah, lo: terah, hi: terah) -> terah {
    nagah sanna value < lo {
        yuxadalo lo
    }
    nagah sanna value > hi {
        yuxadalo hi
    }
    yuxadalo value
}

fnc kort() {
    yazde("{clamp(5, 0, 10)}")    // 5
    yazde("{clamp(-3, 0, 10)}")   // 0
    yazde("{clamp(42, 0, 10)}")   // 10
}
```

## Имена параметров — не забывай про `a`

Зарезервированные слова (`a`, `ya`, типы, контрольные слова) нельзя использовать как имена параметров. Классическая ошибка:

```mott
fnc add(a: terah, b: terah) -> terah {   // ошибка: `a` это AND-оператор
    yuxadalo a + b
}
```

Правильно — любое другое имя:

```mott
fnc add(x: terah, y: terah) -> terah {
    yuxadalo x + y
}
```

## Пример: программа из нескольких функций

```mott
fnc square(x: terah) -> terah {
    yuxadalo x * x
}

fnc sum_of_squares(n: terah) -> terah {
    xilit total: terah = 0
    yallalc i chu 1..(n + 1) {
        total = total + square(i)
    }
    yuxadalo total
}

fnc kort() {
    yazde("sum of 1²..10² = {sum_of_squares(10)}")
}
// sum of 1²..10² = 385
```

Обрати внимание как функции собираются: `kort` использует `sum_of_squares`, которая использует `square`. Каждая функция делает свою задачу, `kort` только дирижирует.

## Попробуй сам

1. Напиши функцию `max(x: terah, y: terah) -> terah`, возвращающую большее из двух. Проверь на нескольких парах.
2. Напиши `pow(base: terah, exp: terah) -> terah` — возводит `base` в степень `exp` (через цикл). `pow(2, 10)` должно дать 1024.
3. Напиши рекурсивную функцию `fib(n: terah) -> terah` — n-е число Фибоначчи. `fib(10)` должно быть 55.
4. Напиши функцию `describe(n: terah)` (без возврата), которая печатает "positive", "zero" или "negative" в зависимости от знака.
