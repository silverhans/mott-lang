# Глава 2. Переменные и типы

## Объявление через `xilit`

Переменная в Mott объявляется ключевым словом **`xilit`** (чеч. "держи", "будь"). Простейший вид:

```mott
fnc kort() {
    xilit x = 5
    yazde(x)            // -> 5
}
```

`xilit` — создаёт **новую** переменную. Это примерно как `let` в Rust/JavaScript или `var` в Go.

## Четыре примитивных типа

В Mott (на момент v0.2) четыре базовых типа:

| Тип | Что это | Примеры литералов |
|---|---|---|
| `terah` | 64-битное целое (int64) | `42`, `-7`, `0`, `9999999` |
| `daqosh` | 64-битное с плавающей точкой (double, IEEE 754) | `3.14`, `-0.5`, `2.0` |
| `bool` | логический | `baqderg` (true), `xarco` (false) |
| `deshnash` | строка UTF-8 | `"salam"`, `"hello {name}"` |

Чеченские корни:
- **`terah`** — "число, цифра"
- **`daqosh`** — "кусочек, долька" (намёк на "дробный")
- **`deshnash`** — "слова", множественное от *дош* (слово)
- **`baqderg`** — "истинное, правдивое"
- **`xarco`** — "ложное, отрицание"

## Вывод типа

Компилятор сам понимает тип по значению справа:

```mott
xilit n = 42              // -> terah
xilit pi = 3.14           // -> daqosh
xilit flag = baqderg      // -> bool
xilit s = "salam"         // -> deshnash
```

Удобно для коротких программ: не надо дублировать тип.

## Явный тип

Иногда хочется написать тип явно — для документации, или когда компилятор не может вывести, или когда хочется быть уверенным. Пишется через двоеточие:

```mott
xilit n: terah = 42
xilit pi: daqosh = 3.14
xilit ok: bool = baqderg
xilit greeting: deshnash = "salam"
```

Если значение справа **не совпадает** с указанным типом, компилятор выдаст ошибку:

```mott
xilit x: terah = "not a number"
// error: type mismatch: `x` declared as terah but initializer is deshnash
```

## Когда явный тип обязателен

- **Когда нет значения** (см. ниже). `xilit x: terah` — ок, `xilit x` — ошибка.
- **Хочешь `daqosh` для целого числа**: `xilit y: daqosh = 5` не сработает (справа `5` это `terah`). Придётся `xilit y: daqosh = 5.0` или `xilit y: daqosh = to_daqosh(5)`.
- **Пустой массив**: `xilit nums: [terah] = []` — без аннотации компилятору неоткуда вывести тип элемента.

## Объявление без значения

Можно объявить переменную, не присваивая ей ничего сразу — если указан тип:

```mott
fnc kort() {
    xilit x: terah
    xilit s: deshnash
    xilit flag: bool
    xilit nums: [terah]

    yazde(x)          // 0
    yazde(s)          // пустая строка
    yazde(flag)       // xarco

    x = 42
    yazde(x)          // 42
}
```

Каждый тип имеет своё **zero-value**:
- `terah` → `0`
- `daqosh` → `0.0`
- `bool` → `xarco`
- `deshnash` → `""` (пустая)
- массив `[T]` → пустой массив длины 0

Без типа (`xilit x`) — ошибка компиляции. Компилятор не умеет угадывать тип из воздуха:

```mott
xilit x
// error: variable `x` needs either a type annotation
// (`xilit x: terah`) or an initializer (`xilit x = 0`)
```

## Присваивание

Переменные в Mott **изменяемые** по умолчанию. Значение меняется через `=`, без `xilit`:

```mott
fnc kort() {
    xilit x = 5
    yazde(x)         // -> 5

    x = 10
    yazde(x)         // -> 10

    x = x + 1
    yazde(x)         // -> 11
}
```

Присваивание проверяет тип: `x` это `terah`, значит справа должен быть тоже `terah`. Нельзя:

```mott
xilit x: terah = 5
x = "salam"          // error: type mismatch
```

## Область видимости

Переменная живёт внутри того `{...}` блока, где была объявлена:

```mott
fnc kort() {
    xilit x = 10
    nagah sanna x > 0 {
        xilit y = 20
        yazde(x + y)     // -> 30, обе видны
    }
    // тут y уже не видна
    yazde(x)             // -> 10, x ещё жива
}
```

## Повторное объявление

**Запрещено** в том же блоке:

```mott
fnc kort() {
    xilit x = 5
    xilit x = 6          // error: variable `x` already declared in this scope
}
```

В разных блоках — можно (каждый блок создаёт свою область):

```mott
fnc kort() {
    xilit x = 5
    nagah sanna baqderg {
        xilit x = 100    // ok, это другая `x` внутри блока
        yazde(x)         // -> 100
    }
    yazde(x)             // -> 5, внешняя x не затронута
}
```

## Имена переменных

Правила:
- Начинается с буквы латинского алфавита (`a-z`, `A-Z`) или `_`
- Дальше — буквы, цифры, `_`
- Только ASCII в MVP

```mott
xilit x = 1              // ок
xilit user_count = 10    // ок
xilit _tmp = 0           // ок
xilit Name123 = "Hans"   // ок

xilit 1st = "error"      // ошибка: начинается с цифры
```

## Зарезервированные имена

Некоторые слова — **ключевые**, их нельзя использовать как имена переменных. Полный список в [справочнике](appendix.md), но особое внимание:

- **`a`** — это оператор AND. Нельзя `xilit a = 5`. Используй `alpha`, `x`, `arg`, что угодно другое.
- **`ya`** — это оператор OR. Аналогично.
- **Слова-типы** (`terah`, `bool`, `deshnash`, `daqosh`) — тоже нельзя как имена.

Если случайно напишешь переменную с именем `a` — получишь парсер-ошибку "expected variable name".
