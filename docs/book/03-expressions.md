# Глава 3. Выражения и операторы

В этой главе — всё, что можно вычислить. Арифметика, сравнения, логические операции, и как они все друг с другом сочетаются.

## Арифметика

Обычные математические операторы. Работают на `terah` и `daqosh`:

| Оператор | Что делает |
|---|---|
| `+` | сложение |
| `-` | вычитание (также унарный "минус") |
| `*` | умножение |
| `/` | деление |
| `%` | остаток от деления (только для `terah`) |

Примеры:

```mott
fnc kort() {
    xilit a = 10
    xilit b = 3

    yazde(a + b)         // 13
    yazde(a - b)         // 7
    yazde(a * b)         // 30
    yazde(a / b)         // 3  (целочисленное деление!)
    yazde(a % b)         // 1

    xilit pi = 3.14
    xilit r = 2.0
    yazde(pi * r * r)    // 12.56
}
```

**Деление `terah / terah` — целочисленное**, как в C/Go. `10 / 3` даст `3`, а не `3.33...`. Если нужен дробный результат — работай в `daqosh`.

**`%` работает только на `terah`**. Для `daqosh` остаток смысла не имеет (или имеет, но мы это не реализовали).

## Унарный минус

```mott
xilit x = -42             // ок
xilit y = -x              // ок, минус перед переменной
```

## Сравнения

Все возвращают `bool`:

| Оператор | Что делает |
|---|---|
| `==` | равно |
| `!=` | не равно |
| `<` | меньше |
| `<=` | меньше или равно |
| `>` | больше |
| `>=` | больше или равно |

```mott
fnc kort() {
    xilit x = 5
    yazde(x == 5)         // baqderg (true)
    yazde(x != 5)         // xarco  (false)
    yazde(x < 10)         // baqderg
    yazde(x >= 100)       // xarco
}
```

**`==` и `!=` работают и на строках** — сравнение побайтовое:

```mott
xilit name = "Hans"
yazde(name == "Hans")     // baqderg
yazde(name == "hans")     // xarco (регистр важен)
yazde(name != "Peter")    // baqderg
```

**`<`, `>`, `<=`, `>=` на строках — не работают** (пока). Лексикографическое сравнение не реализовано; MVP бьёт ошибкой.

## Логические операторы

Здесь Mott отличается от привычного. Вместо `&&` и `||` используются **чеченские частицы**.

### AND — постфиксное `a`

В чеченском языке "и" присоединяется к каждому члену как суффикс/частица. Mott повторяет этот паттерн. Синтаксис:

```
expr a, expr a
```

После каждого конъюнкта ставится `a`, между ними — запятая. **Минимум два конъюнкта**.

```mott
fnc kort() {
    xilit x = 5
    nagah sanna x > 0 a, x < 10 a {
        yazde("x в диапазоне (0, 10)")
    }

    // три условия
    nagah sanna x > 0 a, x < 100 a, x != 42 a {
        yazde("x в диапазоне и не 42")
    }
}
```

**Важная особенность парсера:** `a` "жадный". Как только парсер увидит `a` после выражения, он ожидает продолжение `, expr a`. Это создаёт конфликт с запятой-разделителем в вызовах функций:

```mott
yazde(x a, y a)          // ошибка: парсер думает это 2-конъюнктный AND как аргумент, но `)` не закрывает AND правильно
yazde((x a, y a))        // ok: внешние скобки изолируют AND как выражение
```

Внутри условий `nagah sanna` / `cqachunna` / `yallalc` — проблемы нет: открывающая `{` чётко завершает AND-цепочку.

### OR — инфиксное `ya`

OR проще — между выражениями:

```
expr ya expr
```

Может быть n-арным:

```mott
fnc kort() {
    xilit x = 5
    nagah sanna x == 0 ya x == 5 ya x == 10 {
        yazde("x — 0, 5 или 10")
    }
}
```

### NOT — унарное `!`

```mott
fnc kort() {
    xilit ready = xarco
    nagah sanna !ready {
        yazde("не готово")
    }
}
```

### Смешивание AND и OR

AND связывает сильнее OR. То есть `a ya b a, c a` читается как `a ya (b a, c a)`:

```mott
nagah sanna x == 0 ya x > 0 a, x < 10 a {
    // (x == 0) OR ((x > 0) AND (x < 10))
    yazde("подходит")
}
```

Если хочешь другую группировку — расставь скобки.

## Приоритеты операторов

От самого высокого к самому низкому:

| Приоритет | Операторы | Тип |
|---|---|---|
| 1 (самый высокий) | унарный `-`, унарный `!` | |
| 2 | `*`, `/`, `%` | left-assoc |
| 3 | `+`, `-` | left-assoc |
| 4 | `==`, `!=`, `<`, `<=`, `>`, `>=` | left-assoc |
| 5 | AND с постфиксным `a` | n-ary |
| 6 | OR с `ya` | n-ary |
| 7 (самый низкий) | `=` присваивание (только как statement) | |

Знакомо — такой же порядок как в C/Rust/большинстве языков, просто логические операторы записаны по-другому.

## Скобки для группировки

Обычные круглые скобки работают как везде:

```mott
xilit result = (a + b) * c          // сначала сумма, потом умножить
nagah sanna (x > 0 ya y > 0) a, z != 0 a {
    // ...
}
```

## `=` это не выражение

В отличие от C, **присваивание — не выражение**. Нельзя:

```mott
nagah sanna (x = 5) { ... }    // ошибка парсера
```

Это специально. Частый баг в C — перепутать `=` с `==`. Mott физически не даёт его совершить: `x = 5` это **инструкция**, не значение.
