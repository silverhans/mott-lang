# Глава 7. Ввод и вывод

До этого твои программы были монологами — только `yazde`. В этой главе научимся получать данные от пользователя через `esha` и делать интерактивные программы.

## `yazde` — вывод

Ты уже им пользовался. Краткая сводка:

```mott
yazde("строка")            // печатает строку + \n
yazde(42)                  // печатает число + \n
yazde(3.14)                // -> 3.14
yazde(baqderg)             // -> baqderg
```

`yazde` — **встроенный**, работает на любом примитивном типе (`terah`, `daqosh`, `bool`, `deshnash`). Автоматически добавляет перевод строки в конце.

### Для массивов yazde не работает напрямую

```mott
xilit nums: [terah] = [1, 2, 3]
yazde(nums)                                    // ошибка
```

Компилятор скажет: `can't print arrays directly yet — iterate with yallalc and print each element`. Правильный путь — цикл:

```mott
yallalc n chu nums {
    yazde(n)
}
```

### `yazde` и интерполяция

Удобная идиома — один `yazde` со всеми значениями через `{...}`:

```mott
xilit name: deshnash = "Hans"
xilit score: terah = 42
yazde("{name} набрал {score} очков.")
```

Вместо трёх отдельных `yazde` или конкатенации.

## `esha` — ввод

`esha` (чеч. *"эша"* — "послушай") — читает **одну строку** со стандартного входа. Возвращает `deshnash`.

```mott
fnc kort() {
    yazde("Как тебя зовут?")
    xilit name: deshnash = esha()
    yazde("Salam, {name}!")
}
```

Запусти:
```
$ mott greet.mott -o greet && ./greet
Как тебя зовут?
Hans
Salam, Hans!
```

### Что делает `esha`

- Ждёт, пока пользователь наберёт строку и нажмёт Enter
- Возвращает всё до Enter, **без** самого `\n`
- На EOF (Ctrl+D) возвращает пустую строку `""`

### С-style, не Python

В Python `input("prompt: ")` совмещает вывод приглашения и ввод. В Mott **раздельно**:

```mott
// Mott (как C/Rust):
yazde("Введи имя:")
xilit name = esha()

// НЕ Python-style:
// xilit name = esha("Введи имя:")   // esha не принимает аргументов
```

Это сознательный выбор — `esha` занимается только вводом, ничего лишнего.

### Без промпта на той же строке

Если хочешь чтобы приглашение и ввод были на одной строке — нужна другая техника. Сейчас `yazde` всегда добавляет `\n`, отдельной функции для "печати без перевода строки" в MVP нет. Поэтому:

```
Введи имя:
Hans
```
будет выглядеть двухстрочно. В v0.3 вероятно добавим `yazde_no_newline` или флаг.

## Парсинг ввода

`esha` даёт `deshnash`. Что если тебе нужно число?

Для этого есть две встроенные функции:

- **`parse_terah(s: deshnash) -> terah`** — превращает строку в целое
- **`parse_daqosh(s: deshnash) -> daqosh`** — превращает строку в число с плавающей точкой

```mott
fnc kort() {
    yazde("Сколько тебе лет?")
    xilit age: terah = parse_terah(esha())
    yazde("через 10 лет тебе будет {...}")
    xilit future: terah = age + 10
    yazde("через 10 лет тебе будет {future}")
}
```

Можно и в две строки — сначала `esha`, потом `parse_terah`:

```mott
xilit line: deshnash = esha()
xilit age: terah = parse_terah(line)
```

Идентично по результату. Делай как удобнее.

### Что принимает `parse_terah`

- Целое число в десятичной системе: `42`, `0`, `-7`
- Ведущие пробелы — разрешены (`"  42"` → `42`)
- Знак `+` или `-` перед числом
- **Без** трейлинг-мусора: `"42abc"` не пройдёт

### Что принимает `parse_daqosh`

Все формы `double`:

```
3.14        // обычный
-0.5        // со знаком
1e10        // экспонента
1.5e-3      // и с точкой и с экспонентой
inf         // бесконечность
nan         // not-a-number
```

### Если на входе мусор

```mott
xilit n: terah = parse_terah("hello")
```

Программа **аварийно завершится** с сообщением:

```
mott runtime: parse_terah: not a valid integer: 'hello'
```

Нет пока `Option`/`Result`-типа, поэтому защищаться от невалидного ввода приходится до парсинга (например, сравнивать со списком ожидаемых ответов). Когда такие типы появятся, `parse_*` научатся возвращать ошибку вместо abort'а.

Другие разновидности ошибки:
- Пустая строка: `parse_terah: empty string: ''`
- Переполнение: `parse_terah: out of range: '999999999999999999999'`

### Простое сравнение строк — иногда быстрее

Если вариантов немного, и числа тебе не нужны — не парси, сравнивай:

```mott
fnc kort() {
    yazde("Продолжить? (y/n)")
    xilit answer: deshnash = esha()

    nagah sanna answer == "y" ya answer == "yes" {
        yazde("OK, поехали")
    } vusht {
        yazde("отмена")
    }
}
```

## Интерактивный цикл

Повторяющееся чтение — обычный `cqachunna`:

```mott
fnc kort() {
    yazde("Введи несколько команд. Пустая строка — выход.")
    cqachunna baqderg {
        xilit cmd: deshnash = esha()
        nagah sanna cmd == "" {
            sac
        }
        yazde("ты сказал: {cmd}")
    }
    yazde("пока!")
}
```

## Пайп из шелла

Стандартный ввод можно подать из файла или через пайп:

```sh
$ echo "Hans" | ./greet
Как тебя зовут?
Salam, Hans!

$ ./greet < names.txt
```

Программа не отличает интерактивный ввод от файлового — просто читает stdin.

## Перенаправление вывода

`yazde` пишет в stdout. Значит стандартный редирект из шелла работает:

```sh
$ ./mott-program > output.txt
$ ./mott-program | grep pattern
```

Ошибки компилятора идут в stderr — в `output.txt` не попадут.

## Пример: сумма чисел

Теперь, когда `parse_terah` в наличии, настоящий интерактивный калькулятор:

```mott
fnc kort() {
    yazde("первое число:")
    xilit first: terah = parse_terah(esha())
    yazde("второе число:")
    xilit second: terah = parse_terah(esha())
    xilit sum: terah = first + second
    yazde("сумма = {sum}")
}
```

```
$ ./sum
первое число:
3
второе число:
4
сумма = 7
```

Или через пайп:

```
$ printf "10\n32\n" | ./sum
первое число:
второе число:
сумма = 42
```

## Попробуй сам

1. Интерактивная "угадайка": запроси у пользователя слово. Если это `"salam"` — ответь `"wa aleykum as-salam"`. Если `"marsha"` — ответь `"marsha vogilla"`. Иначе — `"не знаю такого"`.
2. Напиши программу-эхо: читает строку и печатает её обратно. Бесконечный цикл, выход по пустой строке.
3. "Счётчик строк": пусть программа читает строки пока не будет пустой, и печатает общее число строк в конце. (Используй `cqachunna`, переменную-счётчик, `sac`.)
4. **Калькулятор-средний**: запроси три числа, посчитай их сумму и среднее (через целочисленное деление — `daqosh` пока не нужно). Используй `parse_terah(esha())`.
5. **Угадай число**: программа "загадывает" число 42, просит пользователя ввести догадку. Если угадал — `"верно!"`, если меньше — `"больше"`, если больше — `"меньше"`. Цикл до правильного ответа.
6. Что вернёт `esha()`, если запустить программу через `./my-program < /dev/null`? А что будет, если передать это в `parse_terah`?
