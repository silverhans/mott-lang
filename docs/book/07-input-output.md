# Глава 7. Ввод и вывод

До этого твои программы были монологами — только `yazde`. В этой главе научимся получать данные от пользователя через `esha` и делать интерактивные программы.

## `yazde` — вывод

Ты уже им пользовался. Краткая сводка:

```mott
yazde("строка")            // печатает строку + \n
yazde(42)                  // печатает число + \n
yazde(3.14)                // -> 3.14
yazde(baqderg)             // -> baqderg
```

`yazde` — **встроенный**, работает на любом примитивном типе (`terah`, `daqosh`, `bool`, `deshnash`). Автоматически добавляет перевод строки в конце.

### Для массивов yazde не работает напрямую

```mott
xilit nums: [terah] = [1, 2, 3]
yazde(nums)                                    // ошибка
```

Компилятор скажет: `can't print arrays directly yet — iterate with yallalc and print each element`. Правильный путь — цикл:

```mott
yallalc n chu nums {
    yazde(n)
}
```

### `yazde` и интерполяция

Удобная идиома — один `yazde` со всеми значениями через `{...}`:

```mott
xilit name: deshnash = "Hans"
xilit score: terah = 42
yazde("{name} набрал {score} очков.")
```

Вместо трёх отдельных `yazde` или конкатенации.

## `esha` — ввод

`esha` (чеч. *"эша"* — "послушай") — читает **одну строку** со стандартного входа. Возвращает `deshnash`.

```mott
fnc kort() {
    yazde("Как тебя зовут?")
    xilit name: deshnash = esha()
    yazde("Salam, {name}!")
}
```

Запусти:
```
$ mott greet.mott -o greet && ./greet
Как тебя зовут?
Hans
Salam, Hans!
```

### Что делает `esha`

- Ждёт, пока пользователь наберёт строку и нажмёт Enter
- Возвращает всё до Enter, **без** самого `\n`
- На EOF (Ctrl+D) возвращает пустую строку `""`

### С-style, не Python

В Python `input("prompt: ")` совмещает вывод приглашения и ввод. В Mott **раздельно**:

```mott
// Mott (как C/Rust):
yazde("Введи имя:")
xilit name = esha()

// НЕ Python-style:
// xilit name = esha("Введи имя:")   // esha не принимает аргументов
```

Это сознательный выбор — `esha` занимается только вводом, ничего лишнего.

### Без промпта на той же строке

Если хочешь чтобы приглашение и ввод были на одной строке — нужна другая техника. Сейчас `yazde` всегда добавляет `\n`, отдельной функции для "печати без перевода строки" в MVP нет. Поэтому:

```
Введи имя:
Hans
```
будет выглядеть двухстрочно. В v0.3 вероятно добавим `yazde_no_newline` или флаг.

## Парсинг ввода

`esha` даёт `deshnash`. Что если тебе нужно число?

В MVP **нет встроенной функции** `parse_terah`. Компилятор умеет превращать числа в строки (интерполяция), но не обратно. Это запланировано на v0.3 стандартной библиотеки.

Пока — обрабатывай строки, либо сравнивай с известными вариантами:

```mott
fnc kort() {
    yazde("Продолжить? (y/n)")
    xilit answer: deshnash = esha()

    nagah sanna answer == "y" ya answer == "yes" {
        yazde("OK, поехали")
    } khi {
        yazde("отмена")
    }
}
```

## Интерактивный цикл

Повторяющееся чтение — обычный `cqachunna`:

```mott
fnc kort() {
    yazde("Введи несколько команд. Пустая строка — выход.")
    cqachunna baqderg {
        xilit cmd: deshnash = esha()
        nagah sanna cmd == "" {
            sac
        }
        yazde("ты сказал: {cmd}")
    }
    yazde("пока!")
}
```

## Пайп из шелла

Стандартный ввод можно подать из файла или через пайп:

```sh
$ echo "Hans" | ./greet
Как тебя зовут?
Salam, Hans!

$ ./greet < names.txt
```

Программа не отличает интерактивный ввод от файлового — просто читает stdin.

## Перенаправление вывода

`yazde` пишет в stdout. Значит стандартный редирект из шелла работает:

```sh
$ ./mott-program > output.txt
$ ./mott-program | grep pattern
```

Ошибки компилятора идут в stderr — в `output.txt` не попадут.

## Пример: сумма чисел

Поскольку парсинга чисел пока нет, давайте сделаем что-то на строках:

```mott
fnc kort() {
    yazde("Введи два слова — верну их вместе:")
    xilit a: deshnash = esha()
    xilit b: deshnash = esha()
    yazde("объединение: {a} и {b}")
}
```

```
$ ./concat
Введи два слова — верну их вместе:
marsha
vogilla
объединение: marsha и vogilla
```

## Попробуй сам

1. Интерактивная "угадайка": запроси у пользователя слово. Если это `"salam"` — ответь `"wa aleykum as-salam"`. Если `"marsha"` — ответь `"marsha vogilla"`. Иначе — `"не знаю такого"`.
2. Напиши программу-эхо: читает строку и печатает её обратно. Бесконечный цикл, выход по пустой строке.
3. "Счётчик слов": пусть программа читает строки пока не будет пустой, и печатает общее число строк в конце. (Используй `cqachunna`, переменную-счётчик, `sac`.)
4. Что вернёт `esha()`, если запустить программу через `./my-program < /dev/null`?
