# Глава 4. Управление потоком

Условия, циклы, выход из циклов. Mott даёт базовый набор: `if/else/else-if`, `while`, `for-each`, `break`, `continue`.

## `nagah sanna` — условие (if)

Буквально: **"нагахь санна"** — "если так". Лексер склеивает двусловную пару в один токен на этапе сканирования, так что ты всегда пишешь эту фразу целиком.

```mott
fnc kort() {
    xilit x = 5
    nagah sanna x > 0 {
        yazde("положительное")
    }
}
```

**Обрати внимание:** скобок вокруг условия нет. В Mott `if` / `while` / `for` обходятся без них — открывающая `{` однозначно отделяет условие от тела. Скобки внутри выражения, конечно, можно (для приоритетов).

## `vusht` — иначе (else)

`vusht` (чеч. "другое") — ветка которая выполняется если условие ложно:

```mott
fnc kort() {
    xilit x = 5
    nagah sanna x > 10 {
        yazde("большое")
    } vusht {
        yazde("не очень большое")
    }
}
```

`vusht`-ветка опциональна. Без неё просто "если истина — выполни, иначе ничего".

## Цепочки `vusht nagah sanna`

Несколько условий подряд записываются элегантной цепочкой:

```mott
fnc kort() {
    xilit x = 5
    nagah sanna x < 0 {
        yazde("отрицательное")
    } vusht nagah sanna x == 0 {
        yazde("ноль")
    } vusht nagah sanna x < 10 {
        yazde("маленькое")
    } vusht {
        yazde("большое")
    }
}
```

Для компилятора это **синтаксический сахар** — внутри он распаковывается во вложенные `nagah sanna` внутри `vusht`-блоков. Но в сгенерированном C получается идиоматичная `else if` цепочка, а не ёлочка из скобок. Читается ровно так, как пишется.

Финальный `vusht { ... }` опционален.

## `vusht` на новой строке тоже работает

Если тебе удобнее разбить на несколько строк — можно:

```mott
nagah sanna x == 1 {
    yazde("one")
}
vusht nagah sanna x == 2 {
    yazde("two")
}
vusht {
    yazde("other")
}
```

Парсер пропускает автоматически вставленный на переводе строки терминатор перед `vusht`. Удобно для длинных цепочек.

## `cqachunna` — цикл while

От чеченского *"цкъачунна"* (пока / до тех пор пока). Синтаксис знакомый:

```mott
fnc kort() {
    xilit i = 0
    cqachunna i < 5 {
        yazde("{i}")
        i = i + 1
    }
}
// 0
// 1
// 2
// 3
// 4
```

Скобок вокруг условия снова нет — так же как у `nagah sanna`.

**Бесконечный цикл:**

```mott
cqachunna baqderg {
    // работает пока не встретит sac
}
```

## `yallalc ... chu ...` — цикл for-each

Для итерации по массивам и диапазонам в Mott есть отдельный цикл. Полное введение в массивы — в [главе 8](08-arrays.md), а здесь — ознакомление с циклом.

**Итерация по диапазону** (`0..n` — полуоткрытый, `n` не включается):

```mott
yallalc i chu 0..5 {
    yazde("{i}")
}
// 0 1 2 3 4
```

`chu` — чеченское "в, внутри", играет роль английского `in`. Читается как "для каждого `i` в диапазоне 0 до 5".

**Итерация по массиву:**

```mott
xilit names: [deshnash] = ["Hans", "Mansur", "Timur"]
yallalc name chu names {
    yazde(name)
}
```

## `sac` — break

От чеченского *"сацо"* (остановиться). Немедленный выход из ближайшего охватывающего цикла (`cqachunna` или `yallalc`):

```mott
fnc kort() {
    xilit i = 0
    cqachunna baqderg {
        nagah sanna i >= 5 {
            sac                  // выйти из цикла
        }
        yazde("{i}")
        i = i + 1
    }
    yazde("done")
}
```

## `khida` — continue

От чеченского *"хьида"* (иди/двигайся дальше). Пропустить остаток тела и перейти к следующей итерации:

```mott
fnc kort() {
    xilit i = 0
    cqachunna i < 10 {
        i = i + 1
        nagah sanna i % 2 == 0 {
            khida                // пропустить чётные
        }
        yazde("{i}")
    }
}
// 1 3 5 7 9
```

## Вне цикла `sac` и `khida` — ошибка

```mott
fnc kort() {
    sac                          // error: `sac` outside of `cqachunna` loop
}
```

Компилятор ловит это до того как C увидит код. Также работает внутри `nagah sanna` внутри цикла — там `sac` относится к ближайшему циклу:

```mott
cqachunna i < 10 {
    nagah sanna found {
        sac                      // ок — выходит из cqachunna
    }
    i = i + 1
}
```

## Вложенные циклы

Работают как ожидаешь; `sac` выходит из **ближайшего** цикла:

```mott
yallalc i chu 0..3 {
    yallalc j chu 0..3 {
        nagah sanna j == 2 {
            sac                  // выход только из внутреннего
        }
        yazde("{i},{j}")
    }
}
// 0,0 0,1 1,0 1,1 2,0 2,1
```

Мульти-уровневый `break` (как в Go через метки) — не поддерживается в MVP. Если нужно — обходи флагом.

## Пример: FizzBuzz

Классика, в которой есть всё:

```mott
fnc kort() {
    xilit i: terah = 1
    cqachunna i <= 100 {
        nagah sanna i % 3 == 0 a, i % 5 == 0 a {
            yazde("FizzBuzz")
        } vusht nagah sanna i % 3 == 0 {
            yazde("Fizz")
        } vusht nagah sanna i % 5 == 0 {
            yazde("Buzz")
        } vusht {
            yazde("{i}")
        }
        i = i + 1
    }
}
```

Заметь: `i % 3 == 0 a, i % 5 == 0 a` — это AND двух условий, чеченская пост-фиксная запись.

## Попробуй сам

1. Напиши цикл, который печатает числа от 1 до 20, но заменяет числа кратные 3 на "Fizz" (без Buzz, облегчённая версия).
2. Программа: ввести в `xilit n: terah = 17` и найти все его делители (числа от 1 до n, которые делят его без остатка). Напечатать их.
3. Вложенные циклы: напечатай таблицу умножения 5×5, форматом `2 * 3 = 6`. Используй `yallalc` и range.
4. Что произойдёт, если в `cqachunna baqderg` не будет `sac`? Запусти, останови через `Ctrl+C`, подумай почему.
